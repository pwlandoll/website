2014/03/08: The mid-sized program is complete! pigLatin.boo prompts the user for the name of a text file, reads in the contents, and translates all the translatable words in the file. It then prompts the user for the name of the output file, then writes the translated contents, creating a file if it has to. It consists of four methods and a main method. readInFile() prompts the user for a file name and returns the contents of the file as a string. firstVowel() takes in a string as a parameter and returns the position of the first vowel as an integer. pig() takes in a string and an integer as parameters and returns a reordering (according to the rules of Pig Latin) of the string dependent on the integer. writeOutFile() takes in a string as a parameter and prompts the user for a file name, then writes the contents of the string to a file of the specified name.

There are some great things about Boo that actually made this a lot easier than I had anticipated. Some of the references I had found before were actually somewhat useless. For starters, anything reliant on the Useful.IO library wasn't helpful. I wasn't able to successfully import Useful so I had to look for other options. The "using" keyword didn't seem to help much for file content manipulation, as I had hoped it would. What ended up being the most helpful was the fact that Boo is based on .NET, so I tried using reference pages from Microsoft to find ideas. As it turns out I could use methods from .NET's File class, and my task was made much easier.

I also got to learn a little bit of regular expressions. In the main method, the string of contents from the file is split by whitespace, which was easily done with a regular expression. I also used a regex to check if a word was eligible for translation, meaning it matched the pattern of being only letters. I also used regular expressions to replace a word in the contents string with its translation. Making a regular expression out of a string is as simple as saying Regex(string).
